# Anti-Slop

These patterns read as machine-generated regardless of language or channel. Avoid all of them in every draft.

- **No "not X, but Y" / "rather than Y" antithesis.** Don't set up a rejected alternative just to land on the real point ("I'd rather ask a couple of things than send you a long list", "the one or two that actually make sense rather than everything on the board"). State the real point directly. This construction is fine to use once in a long piece of writing; appearing twice in one short message is the tell.
- **No hollow intensifiers.** Never use "actually" (or the equivalent — "eigentlich", "на самом деле", "właściwie") as a filler emphasis word. If a sentence works without it, it wasn't needed.
- **No stock phrases.** "worth your time", "at your convenience", "don't hesitate to reach out", and similar filler courtesies are clichés, not communication. Say the specific thing instead, or cut it.
- **Don't announce the message's own next move.** Never write "Once I know that, I can point you to..." or "Let me..." or "I'll then be able to..." — describing what you're about to do instead of doing it is a stalling pattern. Ask the question and stop; the next message can do the next thing.
- **At most one em-dash aside per message.** A dash-interruption can land well once. Two or more in a short message reads as a template, not a person typing.
- **Question count is set by lead state, not by default.** See `by-state.md` for exactly how many questions this specific state allows (usually one; NEW allows up to three, each with its own stated reason). Never exceed that count regardless of how much unclear information there is — the rest can wait for the reply.
